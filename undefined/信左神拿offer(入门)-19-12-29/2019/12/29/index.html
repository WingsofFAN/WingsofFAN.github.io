基础概念

时间复杂度

  • 常数操作：一个操作如果和样本的数据量没有关系，每次都是固定时间内完成的操作。

    如从数组中拿出某个元素是常数操作，而从列表中拿出某个元素不是常数操作。

  • 时间复杂度(粗略认识)：常数操作表达式中，只看高阶项，且忽略系数后所留下的东西。

    为什么忽略系数和低阶项？因为当数据量大到一定规模时，起到绝对影响的就是留下的高阶项表达式。（一定是考虑最差情况）

    最差复杂 O()，平均复杂Θ()，最好复杂Ω()

    当时间复杂度相同时，只能通过实际测试去比较。系数所代表的常数操作的固定时间有可能不同。评价一个算法流程的好坏，先看时间复杂度的指标，然后再分析不同数据样本下的实际运行 时间，也就是“常数项时间“。

而外空间复杂度

  • 而外空间：用于辅助算法流程时，新申请的变量空间
  • 而外空间复杂度：而外空间表达式中，只看高阶项，且忽略系数后所留下的东西。

简单排序算法

选择排序

  • 遍历一遍数组中所有的数（每遍历一个数算是一次常数操作）；
  • 遍历时进行比较（遍历一个数就比一次，也是常数操作），找到最小元素；
  • 让最小元素和第一个元素进行交换；
  • 重复上述三个过程找到次小元素，次次小元素……直到数组结束。

遍历了(n)+(n-1)+(n-2)+…+1个元素，比较了(n)+(n-1)+(n-2)+…+1次，交换了n次；所以一共进行了（ann+bn+c）次常数操作，那么时间复杂度就是O(nn)。因为只用到了有限的几个变量，而外空间复杂度为O(1)。

冒泡排序

  • 依次遍历比较两个相邻元素
  • 若大的在前，则和其后的小元素进行交换
  • 遍历一遍后，最小元素已被排到第一的位置，从位置二开始重复上述两步，依次递减处理长度，直到剩下一个元素时。

时间复杂度O(nn),空间复杂度O(1)

插入排序

  • 依次遍历元素，若元素小于其前面的有序序列
  • 则让该元素和其前面的元素依次交换，直到他不小于其前一个元素
  • 此时，有序序列长度增加一，并从有序序列的下一个元素开始重复第一步
  • 直到遍历完所有元素

时间复杂度：数据状况会影响到处理时间，最差情况为O(nn),最佳情况O(n),但是时间复杂度只考虑遇到的最差情况，所以为O(nn)

//插入排序实现
template<typename T>
void InsertSort(vector<T>& nums)
{
    if(nums.size() < 2)
    {    
        return;
    }
    for(int i = 1 ;i < nums.size();i++)
    {
        for(int j = i ; j>0 && nums[j]<nums[j-1];j--)

            swap(nums,j,j-1);

异或

异或运算：相同为零，不同为一。二进制的无进位相加。

用于交换

性质：

// 1. 0^N =N; N^N = 0
// 2. a^b = b^a;(a^b^c) = a^(b^c) 
// 3. 所有元素用^运算连接时，无先后顺序的区别

//用于交换操作，无需使用而外的变量
// a = 甲   b = 乙
void swap(a, b)
{
    a = a ^ b;     //a = 甲^乙  b = 乙
    b = a ^ b;     //a = 甲^乙  b = 甲^乙^乙 = 甲^0 = 甲
    a = a ^ b;     //a = 甲^乙^甲 = 甲^甲^乙 = 0^乙 = 乙 b = 甲
}

使用前提：a与b在内存中是独立的，即要有两块独立的内存。否则会将该位置的内存清零。

面试真题

//1.一种数出现了奇数次，其他数出现偶数次 
//2.两种数出项了奇数次，其他数出现偶数次 
//要求时间O(n),空间O(1)
//
vector<int> find_odd(vector<int>& nums)
{
    if(nums.empty())
        return NULL;

    int f = 0;
    for(int i = 0;i < nums.size();i++)
    {
        f ^= nums[i];
    }
    vector<int> R;
    R.push_back(f);
    return R;
}

vector<int> find_two_odd(vector<int>& nums)
{
    if(nums.empty())
        return NULL;
    vector<int> R;
    int f = 0;
    for(int i = 0;i < nums.size();i++)
    {
        f ^= nums[i]; 
    }
    //此处得到的是a^b

    //因为a，b不相等，所以a^b ≠ 0
    //那么必定存在a与b在某一位上不同，取出该位用于区分a与b
    int right = f & (~f +1);
    //即f中从优往左数的第一个1
    int k = f;
    for(int i = 0;i < nums.size();i++)
    {
        if(nums[i]&right == 0)
            k ^= nums[i];               
    }
    //此处得到a（或b）
    R.push_back(k);
    R.push_back(f^k);
    return R;
}

对数器

方法a：考虑各方面优化，不好想，也不好写，容易出错的方法。

方法b：不考虑优化，很好想，也很好写，一定不会出错的方法。

此时生成一个随机样本产生器，将生成的数据分别运用到方法a与方法b，得到结果a与结果b。

先控制样本数量，采用人工推理来检查方法b与方法a，先确保方法b不出错，再修正方法a。

若结果a和结果b在大量测试样本下都相等，则方法a正确可行。

template<typename T>
void swap(std::vector<T>& vec, int i, int j) {
    vec[i] = vec[i] ^ vec[j];
    vec[j] = vec[i] ^ vec[j];
    vec[i] = vec[i] ^ vec[j];
}

// 随机序列生成器
vector<int> generateRandomArray(int maxSize, int& size, int minValue, int maxValue) {
    assert(minValue < maxValue);
    size = e() % maxSize + 1;
    vector<int> list(size);
    for (int i = 0; i < list.size(); i++) {
        list[i] = e() % (maxValue - minValue) + minValue;
    }
    return list;
}
